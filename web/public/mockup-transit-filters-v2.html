Drill-down behavior:
  • Click a line pill to select it. A small downward chevron appears below selected pills.
  • A summary tag appears (e.g., L · 2 stops). Click the tag to expand/collapse the stop list.
  • Default state when first selecting a line: All stops checked. User unchecks stops they don't want.
  • If all stops are checked, chip reads "L train". If some, it reads "L (Bedford, 1st)".
  • Select All / Clear toggle at top of stop list for quick batch operations.
  • Stop list is scrollable (max 180px) for lines with many stops.
  • Transfer indicators (colored dots) show connecting lines at each stop for context.
  • Multiple lines can each have their own expanded stop panel (e.g., L + G both open).

Chip label logic:
  • All stops on 1 line: "L train"
  • Some stops on 1 line: "L (Bedford, 1st)" (up to 2 names, then "+ N more")
  • Multiple lines, all stops: "L + G trains"
  • Multiple lines, some stops: "L (2) + G" (parenthetical count for filtered lines)
  • Too many rules to fit: "3 commute rules"
