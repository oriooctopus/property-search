Pros

  • Discoverable — always visible in the filter bar alongside other controls
  • Logical placement: "I set my filters, now I save them" left-to-right flow
  • Minimal UI footprint when not active (just one small chip)
  • Consistent with the existing filter chip pattern users already know
  • Separator visually groups it as a distinct action, not a filter

Cons

  • Filter bar can get crowded on narrow viewports with many chips
  • Could be mistaken for a filter if the separator is subtle
  • Dropdown may get clipped on smaller screens if bar is near edge

Pros

  • Prominent and unmissable — floats above content with green CTA styling
  • Only appears when relevant (filters are active), reducing noise
  • Keeps the filter bar clean — no extra chip to compete for space
  • Centered modal feels more intentional and focused than an inline dropdown
  • Works well on mobile — FAB is a familiar mobile pattern

Cons

  • FAB can overlap listing cards or map controls in the bottom-right
  • Modal is more disruptive — blocks the view entirely
  • Animation in/out adds complexity and potential jank
  • Less discoverable than Flow A (users must scroll down or have filters active to see it)
  • May conflict with other floating elements (chat button, scroll-to-top, etc.)
