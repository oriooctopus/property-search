Proposed Unified Flow

One scripts/ingest.ts entry point with composable phases. Daily and full modes share the same code. Enrichment is mandatory and failures are loud.

flowchart TD
  classDef phase fill:#0a1929,stroke:#58a6ff,color:#58a6ff;
  classDef store fill:#0d2818,stroke:#3fb950,color:#3fb950;

  CRON["Single GH Action cron
(daily + weekly full)"]:::phase
  ENT["scripts/ingest.ts
--mode=daily|full
--sources=...
--skip-enrichment
--dry-run --since=..."]:::phase

  CRON --> ENT

  subgraph Phases["Composable phases"]
    P1["1 · fetch (per-source adapters)"]:::phase
    P2["2 · validate (pipeline.ts)"]:::phase
    P3["3 · dedup (in-batch + cross-batch)"]:::phase
    P4["4 · upsert (listings)"]:::phase
    P5["5 · enrich:isochrones"]:::phase
    P6["5 · enrich:year_built (PLUTO)"]:::phase
    P7["5 · enrich:photos (per-source backfill)"]:::phase
    P8["6 · stale-cleanup"]:::phase
    P9["7 · integrity-report"]:::phase
  end

  ENT --> P1 --> P2 --> P3 --> P4 --> P5 --> P6 --> P7 --> P8 --> P9

  LIST[("listings")]:::store
  ISO[("listing_isochrones")]:::store
  SF[("source_freshness")]:::store

  P4 --> LIST
  P5 --> ISO
  P6 --> LIST
  P7 --> LIST
  P4 --> SF
  P9 -. exits non-zero on missing year_built / missing geo / dropped rows .-> ENT
